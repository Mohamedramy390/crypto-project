## PHASE 2 — ECDH (P-256) Key Exchange + KDF

### Alice generates her ECDH keypair on P-256

```
Alice private scalar (secret!)  : <random 256-bit integer in [1, n-1]>
Alice public key Qx (→ Bob)     : a_scalar · Gx  (x-coord of point on P-256)
Alice public key Qy             : a_scalar · Gy  (y-coord of point on P-256)
```

### Bob generates his ECDH keypair on P-256

```
Bob private scalar (secret!)    : <random 256-bit integer in [1, n-1]>
Bob public key Qx (→ Alice)     : b_scalar · Gx
Bob public key Qy               : b_scalar · Gy
```

### Both compute the SAME shared EC point

```
Alice computes: a · Q_B = a · (b·G) = ab·G  → x-coordinate extracted
Bob   computes: b · Q_A = b · (a·G) = ab·G  → x-coordinate extracted
```

> ✓ Shared x-coords **MATCH** — an eavesdropper cannot compute `scalar·G` without the private key
> (Elliptic Curve Discrete Logarithm Problem, ECDLP).

### KDF: HKDF-MD5 — Deriving symmetric keys from shared x

The 256-bit shared x-coordinate is passed through a two-step HKDF-like KDF:

```
Extract: PRK = HMAC-MD5(salt="SecureVault-ECDH-Salt-v1", IKM=shared_x)
Expand : T(i) = HMAC-MD5(PRK, T(i-1) || info || i)
         OKM  = T(1) || T(2) ... (first 32 bytes taken)

Twofish key (16 bytes) : <first 16 bytes of OKM>
RC4     key (16 bytes) : <last  16 bytes of OKM>
```

> HKDF separates the raw shared secret from the key material, ensuring
> statistical independence between different derived keys.

```

```

> Each 16-byte block is processed through an 8-round Feistel network using
> key-dependent S-boxes, MDS matrix multiplication, and PHT.

✓ Twofish encryption done — inner layer applied.

### Step 2 — Outer Layer: RC4 Stream Cipher

```
Twofish CT → 5f8cd2b28aab9266d5d9ed41fef23244
              0e29eec2137f890e330fd8dc8d0d6a8d

RC4 CT     → a78132352f965b2a6b47419153669e27
              208713c5ee827c8a57738ace79611710
```

> RC4 generates a pseudorandom keystream via KSA + PRGA, then XORs it with
> the Twofish ciphertext byte-by-byte.

✓ RC4 encryption done — outer layer applied (double encryption).

### Step 3 — Integrity Tag: MD5 Digest

```
MD5(RC4_CT) : e5a8b75d29a57b8bb61295b802262f09
```

> MD5 is computed over the final RC4 ciphertext. Any single flipped bit
> in transit will produce a completely different 128-bit hash (avalanche effect).

✓ MD5 tag generated and attached to the packet.

### Step 4 — Key Wrapping: ElGamal

Alice encrypts `(Twofish_key || RC4_key)` using **Bob's public key**:

```
c1 = g^k mod p       : 661838794228765914485851724262703108...
c2 = m * y^k mod p   : 503671377003906512586915132611429203...
```

> ElGamal is **probabilistic** — each encryption uses a fresh random `k`.
> Only Bob's private key `x` can recover the bundle via:
> `m = c2 * (c1^x)^(-1) mod p`

✓ Key bundle wrapped with ElGamal — only Bob can open it.
